Необходимо написать модуль, который:
1)	Создаёт таблицу в базе данных для хранения текстовых сообщения с полями:
a.	id (int, autoincrement);
b.	uid (int);
c.	type (int);
d.	title (varchar);
e.	body (text).
2)	Регистрирует сущность  при помощи Entity API для работы с записями данной таблицы.
3)	Определяет страницу с формой добавления записи или редактирования произвольной записи.
4)	Реализует собственный листинг сообщений (не через Views, а при помощи db_select() с extenders) в виде таблицы с возможностью сортировки и c элементом постраничной навигации. По умолчанию активирована сортировка по порядковому номеру записи.
5)	Разрешает доступ к материалам стандартного типа содержимого (Article) на основе того, создавал ли пользователь когда-нибудь текстовое сообщение в таблице из пункта 1.
